# 実装完了ガイド - AI検出と録画機能

## 📋 実装された機能

### 1. 動体検出システム ✅
- **方式**: フレーム差分による動き検出
- **メリット**: 
  - 軽量で高速（メモリ使用量が少ない）
  - ESP32S3で安定動作
  - リアルタイム処理可能
- **動作**: 前フレームとの差分を計算し、5%以上の変化で検出

### 2. 動画録画機能 ✅
- **形式**: MJPEG（Motion JPEG）
- **録画時間**: 30秒間
- **フレームレート**: 10 FPS
- **保存先**: SDカード `/recordings/` ディレクトリ
- **ファイル名**: `rec_YYYYMMDD_HHMMSS.mjpeg`

### 3. ライブストリーミング ✅
- **プロトコル**: MJPEG over HTTP
- **エンドポイント**: `http://[デバイスIP]/stream`
- **形式**: `multipart/x-mixed-replace; boundary=frame`
- **用途**: ブラウザやVLCで直接視聴可能

### 4. Webインターフェース ✅
- リアルタイムシステム状態表示
- 自動更新プレビュー画像（2秒間隔）
- 日本語UI対応
- レスポンシブデザイン

```cpp
// フレーム差分検出

// - 5%以上の変化で動体検出
```

### MJPEG録画フォーマット
```

```

### 状態管理
- **並列処理**: アラート動作中も録画継続
- **状態遷移**: IDLE → DETECTING → ALERT_ACTIVE + RECORDING → IDLE
- **タイミング**: 10秒アラート、30秒録画（並行動作）

## 🎯 使用方法

### 1. 初期セットアップ
```cpp

const char* wifi_ssid = "あなたのWiFi名";
const char* wifi_password = "あなたのパスワード";
```

### 2. ビルドとアップロード
```bash
pio run --target upload
pio device monitor
```

### 3. 動作確認
1. シリアルモニターでIPアドレス確認
2. ブラウザで `http://[IP]/` にアクセス
3. ライブプレビューで映像確認
4. カメラの前で動いて検出テスト

### 4. ストリーミング視聴
- **ブラウザ**: `http://[IP]/stream` を開く
- **VLC**: メディア→ネットワークストリームを開く→URL入力

### SDカード書き込みエラー
